** Installing Dependencies

*** Dependencies with winget

- Needs administrator access when installing packages

- Is installed on recent Windows 11

- Install winget:
  - From the Windows Store: search for "winget"; install the "App Installer" package
  - From github: https://github.com/microsoft/winget-cli, specifically https://github.com/microsoft/winget-cli/releases/latest

#+begin_src powershell
winget install Git.Git
winget install EclipseAdoptium.Temurin.17.JDK
winget install Erlang.ErlangOTP
setx PATH "$env:PATH;C:\Program Files\Erlang OTP\bin"
winget install Microsoft.VisualStudio.2022.BuildTools --override "--add Microsoft.VisualStudio.Workload.VCTools --includeRecommended --add Microsoft.VisualStudio.Component.VC.Tools.x86.x64 --add Microsoft.VisualStudio.Component.Windows11SDK.22000 --passive"
#+end_src

Alternatively, run in an elevated shell:
: & "C:/Program Files (x86)/Microsoft Visual Studio/Installer/vs_installer.exe" modify --installPath "C:\Program Files (x86)\Microsoft Visual Studio\2022\BuildTools" --add Microsoft.VisualStudio.Component.VCTools --passive


Sources:
- [[https://learn.microsoft.com/en-us/visualstudio/install/use-command-line-parameters-to-install-visual-studio?view=vs-2022#use-winget-to-install-or-modify-visual-studio][Use command-line parameters to install Visual Studio | Microsoft Learn]]
- [[https://learn.microsoft.com/en-us/visualstudio/install/workload-and-component-ids?view=vs-2022][Visual Studio workload and component IDs | Microsoft Learn]]

Afterwards all installation commands have completed, open a "Developer PowerShell for VS 2022" tab in Terminal and type:

#+begin_src powershell
git clone https://github.com/abstools/abstools
cd abstools
./gradlew assemble
#+end_src

You will find the generated ~absfrontend.jar~ file in ~frontend/dist/~.

*** DC shutdown

- State machine: running -> shutting down -> stopped

  ABS sets state fron “running” to “shutting down”, backend handles shutdown
  and sets state to “stopped”.

- Valid operations

| State                  | Running | Shutting down | Stopped |
|------------------------+---------+---------------+---------|
| Create objects         | yes     | no            | no      |
| Invoke methods         | yes     | no            | no      |
| Processes keep running | yes     | yes/no        | no      |

  - InfluxDB image doc: [[https://hub.docker.com/r/tutum/influxdb/][https://hub.docker.com/r/tutum/influxdb/]]

  - sample docker-compose.yml:
    https://gist.githubusercontent.com/anonymous/7810ba23b52ba7476c6a/raw/cd25a878bb1dada111f59f6008e1b0e64f76ccb1/docker-compose.yml

#+BEGIN_SRC yaml
InfluxSrv:
  image: "tutum/influxdb:0.8.8"
  ports:
    - "8083:8083"
    - "8086:8086"
  expose:
    - "8090"
    - "8099"
  environment:
    - PRE_CREATE_DB=cadvisor
cadvisor:
  image: "google/cadvisor:0.14.0"
  volumes:
    - "/:/rootfs:ro"
    - "/var/run:/var/run:rw"
    - "/sys:/sys:ro"
    - "/var/lib/docker/:/var/lib/docker:ro"
  links:
    - "InfluxSrv:influxsrv"
  ports:
    - "8080:8080"
  command: "-storage_driver=influxdb -storage_driver_db=cadvisor -storage_driver_host=influxsrv:8086"
grafana:
  image: "grafana/grafana:2.0.2"
  ports:
    - "3000:3000"
  environment:
    - INFLUXDB_HOST=localhost
    - INFLUXDB_PORT=8086
    - INFLUXDB_NAME=cadvisor
    - INFLUXDB_USER=root
    - INFLUXDB_PASS=root
  links:
    - "InfluxSrv:influxsrv"
#+END_SRC
